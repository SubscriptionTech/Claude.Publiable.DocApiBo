# ![Proabono BO](https://www.appvizer.fr/media/application/445/logo/logo-proabono.png)API BO

## __Introduction__

Welcome to the ProAbono Backoffice API documentation.

This API is organized around a RESTful Architecture and an API-First design and designed to help power users and third party integrators to deal with data export and advanced automation capabilities.
Every single functionnality of the ProAbono backoffice can be recoded and rebranded.

## __Specification__

 - This API is documented in **OpenAPI format**
 - /!\ **Upward compatibility** is not guaranteed unless written agreement limited to the required resources.

## __Authentication__

It occurs via **HTTP Basic Auth**. To authenticate your request, you have to add a valid Authorization header.

1. get your API keys from the "integration" section from Proabono.
2. Use the encoded string in the header of your request like in the example below:
```
  POST https://via.proabono.com/Distribution/Customers
  Content-Type: application/json
  Accept: application/json
  Accept-Encoding: gzip, deflate, compress
  Authorization: Basic YzdmYTU3YjgtNDg2MS00ODU3OmY2Nzk3YWYxLTc5MWItNDBiMQ==
```
